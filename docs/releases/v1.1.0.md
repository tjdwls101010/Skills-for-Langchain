This minor release adds a **consultant** behavior on top of the existing evidence-built knowledge skill. The same skill now leads you from an abstract goal to a concrete LangChain/LangGraph/Deep Agents architecture, and — only after you agree — builds it. The measured post-cutoff knowledge is unchanged.

## Install

```bash
claude plugin marketplace add tjdwls101010/Skills-for-Langchain@v1.1.0
claude plugin install skills-for-langchain@skills-for-langchain
```

Restart Claude Code or run `/reload-plugins`. The explicit invocation is `/skills-for-langchain:langchain`.

## Highlights

- **Consultant path (new).** Describe an outcome — "build an agent that triages my support inbox", "automate this weekly report", "answer questions from these PDFs" — even with no framework named and no code shown, and the skill interviews you, proposes a concrete current-API architecture tied to the dimensions it covered, and implements it only on an explicit go-ahead with an agreed scope.
- **Deltas-only path (unchanged).** Editing or reviewing existing LangChain-ecosystem code still gets silent current-API corrections with no interview — the v1.0.0 behavior is preserved.
- **One skill, one knowledge base.** The consultant reads the same two delta references before proposing or coding, so proposals reflect the real current API rather than pre-cutoff reflexes.
- New `references/consultant.md` carries the interview protocol, the ten-dimension checklist, the agreement-gate build rules, and a worked example; the always-loaded `SKILL.md` gained a compact consultant gist and a broadened trigger.

## Validation

- Structural harness validation passed with zero errors and zero warnings.
- The plugin release copy is byte-identical to the canonical skill (`diff -rq` clean).
- Evidence integrity passed: the two delta references are byte-unchanged, and the `SKILL.md` generated-file hash was re-stamped for the consultant edit with an explicit note that the consultant content is not covered by the v1.0.0 probes.
- Claude plugin and marketplace manifests validated in strict mode.
- Consultant behavior was checked by a behavioral dry-run (trigger routing, interview-first posture, reference-reading, and the agreement gate). Consult *quality* is a judgment call reviewed in that dry-run, not a probe-measured number — this is intentionally a lighter bar than the v1.0.0 knowledge probes, because "did it consult well" is not mechanically gradable the way "is this API current" is.

## Compatibility boundary

- Python only.
- The knowledge is verified against an official-documentation snapshot drafted around April 2026 and includes preview, private-preview, beta, and version-gated APIs.
- Review installed package signatures and current official docs for versions substantially newer than the snapshot.
